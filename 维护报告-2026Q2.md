## 二、重要功能交付

### 2.1 业务核心流程优化

| 模块 | 改动内容 |
|------|---------|
| **客户订单** | 新增订单状态手动标记（完成 / 延迟 / 恢复），全屏编辑模式，BOM 下拉搜索，自动带入规格/单位/图片，新建订单后自动创建草稿出货单 |
| **BOM 物料** | 支持图片上传，增加 MOQ 阶梯价编辑，新增认证机构代码与品牌栏位，点击行展开材料明细，SKU 唯一性验证 |
| **采购单（PO）** | 整合收货功能（无需跳转），增加可搜索供应商下拉，支持手动设置单据编号，与 BOM 表关联自动带入料号 |
| **出货单** | 重构为基于客户订单（选客户→选待出货订单→自动带出品项→填出货数量），列表页直接显示确认/出货操作按钮 |
| **生产单** | 完整状态流程（draft → confirmed → shortage/ready → in_progress → completed/cancelled），自动检查库存缺料 |
| **报价单** | 支持 MOQ 阶梯价，支持手动设置报价单编号，优化默认备注文本 |

### 2.2 打印功能完善

- 客户订单打印：新增币种、交货地点、税率/税额/含税合计、预计交货日、负责人、付款方式等打印字段
- 出货单打印：按标准格式重新排版（品名 / 料号 / 规格 / 数量 / 单位 / 单价 / 金额），含公司签章
- 采购单打印：加入 PO 参考号与地址字段
- 统一公司签章管理：所有单据打印自动使用公司设置中的名称与签章，支持可配置签章尺寸
- 草稿状态单据打印时隐藏签章，正式审批后显示

### 2.3 用户体验提升

- **全局可搜索下拉框**：BOM 选择、客户选择、供应商选择全部升级为可搜索过滤模式，弹出方向智能判断，防遮挡
- **悬浮表头**：所有长列表表格支持纵向滚动时表头固定，详情面板内嵌表格表头也同步固定
- **横向滚动**：宽表格支持横向滚动，关键列支持冻结
- **统一状态 UI 组件**：所有订单/单据状态徽章与操作按钮统一为 StatusFlow 组件，视觉风格一致
- **操作确认对话框**：所有状态变更、删除、发送操作前统一弹出确认对话框，防误操作
- **全局 Toast 通知**：错误提示置顶显示，成功提示置底显示，替换原有 `confirm()` 弹窗

### 2.4 系统架构与运维

- **每日自动巡检**：部署 `daily-patrol.yml` 工作流，每日07:00 自动登录系统、调用巡检 API、发送 Telegram 报告，巡检内容包括：订单出货数量异常、缺品项/缺BOM/缺材料、负库存/零库存、各类单据流程卡住等
- **公司信息统一配置**：前台和打印单据的公司名称统一从系统设置读取，不再硬编码
- **角色权限精简**：移除遗留的 legacy admin 角色，统一为 `manager` / `employee` 两级，减少权限混乱
- **部署流程规范化**：明确以 GitHub Actions 为唯一发布渠道，部署完成后 Telegram 通知
